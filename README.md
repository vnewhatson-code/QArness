# QArness

QArness - это harness (набор инструментов) для QA специалистов, помогающий автоматизировать создание QA документации и автотестов.

## Что это такое?

QArness представляет собой коллекцию агентов и навыков (skills) для автоматизации рутинных задач QA инженера:
- Автоматическая генерация тестовой документации
- Создание mind map для тестирования функциональности
- Анализ исходного кода для составления чек-листов
- Документирование форм и интерфейсов

## Основные возможности

### 1. Генерация Mind Map для тестирования

Навык `mindmap-generator` автоматически создает интеллект-карты для тестирования на основе информации о функциональности.

**Что делает:**
- Анализирует описание функциональности из `docs/QA/__workspace_inbox__`
- Анализирует исходный код из последних коммитов
- Составляет структурированный список проверок для QA инженера
- Генерирует MINDMAP.md с полной структурой тестовых проверок
- Создает mindmap.xmind для работы в XMind

**Как использовать:**
1. Поместите файл с описанием функциональности в `docs/QA/__workspace_inbox__`
2. Запросите создание mind map
3. Подтвердите имя папки для функциональности
4. Получите готовые `MINDMAP.md` и `mindmap.xmind` в папке функциональности

**Структура Mind Map включает:**
- Центральный раздел - основная функциональность
- Категории проверок - разбивка по технике действие/параметр/значение
- Общие проверки - проверки всей функциональности целиком
- Проверки по категориям - конкретные тест-кейсы
- Специфические сценарии - граничные случаи и нетривиальные сценарии
- Условия и результаты - ожидаемые исходы проверок

### 2. Документирование форм 1С

Агент `form-documenter` и навык `form-documentation-skill` автоматизируют создание документации для форм 1С.

**Что делает:**
- Анализирует XML-файл формы 1С
- Идентифицирует все элементы интерфейса (кнопки, поля ввода, группы, вкладки)
- Описывает свойства и поведение каждого элемента
- Формирует структурированную документацию в Markdown
- Учитывает события и обработчики формы

**Как использовать:**
1. Проверить наличие недокументированных форм:
   ```bash
   node tools/checkFormsDocumentation.js
   ```
2. Запросить документирование конкретной формы через агента `form-documenter`
3. Получить готовую документацию в `docs/forms/имяФормы.md`

**Выходная документация содержит:**
- Описание всех элементов интерфейса на русском языке
- Информацию о взаимодействии с пользователем
- Свойства и ограничения элементов
- Обработчики событий и действия

## Быстрая установка

QArness устанавливается одной командой и автоматически интегрируется с вашими AI-ассистентами.

### Linux/macOS

```bash
curl -fsSL https://raw.githubusercontent.com/vnewhatson-code/QArness/main/install.sh | bash
```

### Windows (PowerShell)

```powershell
iwr -useb https://raw.githubusercontent.com/vnewhatson-code/QArness/main/install.ps1 | iex
```

### С корпоративным прокси

**Linux/macOS:**
```bash
HTTP_PROXY=http://proxy.company.com:8080 curl -fsSL https://raw.githubusercontent.com/vnewhatson-code/QArness/main/install.sh | bash
```

**Windows:**
```powershell
$env:HTTP_PROXY="http://proxy.company.com:8080"
iwr -useb https://raw.githubusercontent.com/vnewhatson-code/QArness/main/install.ps1 | iex
```

### После установки

Проверьте статус:
```bash
qarness status
```

Интегрируйте с AI-ассистентом:
```bash
qarness integrate opencode
qarness integrate claude
```

📖 **Подробная документация:** [docs/INSTALL.md](docs/INSTALL.md)

## Использование

QArness работает с различными AI-ассистентами (OpenCode, Claude Desktop и др.). После установки используйте навыки и агенты через ваш AI-ассистент:

- Запрашивать создание mind map: "Создай mind map для новой функциональности"
- Документировать формы: "Задокументируй форму [имя_формы]"
- Проверять наличие документации: "Проверь какие формы не задокументированы"

### CLI команды

```bash
qarness update              # Обновить QArness
qarness status              # Показать статус
qarness doctor              # Диагностика проблем
qarness config proxy set    # Настроить прокси
qarness integrate <ai>      # Интегрировать с AI-ассистентом
qarness help                # Показать все команды
```

📖 **Документация CLI:** [docs/CLI.md](docs/CLI.md)

## Требования

### Автоматически устанавливаются

- **Node.js** >= 18.0 (для работы утилит и MCP серверов)
- **npm** (устанавливается с Node.js)
- **XMind MCP server** (для генерации .xmind файлов)

### Требуют ручной установки

- **Git** >= 2.0
- **AI-ассистент** (OpenCode, Claude Desktop или др.)
- **jq** или **Python 3** (для обработки JSON)

## Преимущества использования QArness

1. **Экономия времени** - автоматизация создания документации и тест-планов
2. **Полнота покрытия** - структурированный подход гарантирует учет всех аспектов
3. **Стандартизация** - единый формат документации для всей команды
4. **Анализ кода** - автоматическое составление чек-листов на основе изменений в коде
5. **Гибкость** - легко адаптируется под различные типы функциональности
