# Чеклист тестирования инсталлятора QArness

## Контекст

Инсталлятор (`install.ts`, `uninstall.ts`) и CLI (`src/cli/`) имеют множество кодовых путей: CLI-флаги, интерактивные промпты, файловые операции, rollback, post-install модификации конфигов. Чеклист покрывает все граничные случаи.

Файлы: `install.ts`, `uninstall.ts`, `src/installer/hosts.ts`, `src/installer/features.ts`, `src/installer/manifest.ts`, `src/installer/utils.ts`, `src/installer/cli.ts`, `src/cli/index.ts`, `src/cli/commands/*.ts`

---

## 1. CLI-аргументы и режимы

### 1.1 Базовые флаги
- [ ] `--help` / `-h` → показывает справку и выходит
- [ ] `--yes` / `-y` → авто-выбор всех детектированных хостов и фич
- [ ] `--json` / `-j` → JSON-вывод (подразумевает `--yes`)
- [ ] `--uninstall` / `--remove` → режим удаления

### 1.2 Выбор хостов и фич
- [ ] `--hosts opencode` → установка только в OpenCode
- [ ] `--hosts claude` → установка только в Claude Code
- [ ] `--hosts opencode,claude` → оба хоста
- [ ] `--hosts unknown` → предупреждение + пропуск
- [ ] `--hosts ""` (пустое значение) → пустой список хостов
- [ ] `--features xmind-mcp` → установка конкретной фичи
- [ ] `--features unknown` → предупреждение + пропуск
- [ ] `--features ""` → пустой список фич

### 1.3 Комбинации флагов
- [ ] `--yes --json` → JSON-вывод, авто-выбор
- [ ] `--yes --hosts opencode` → авто-режим, но только указанный хост
- [ ] `--hosts opencode --features xmind-mcp` → без `--yes`, но с предвыбором
- [ ] `--json` без `--yes` → `--yes` подразумевается
- [ ] `--uninstall --yes` → удаление без подтверждения (если реализовано)
- [ ] `--uninstall` без манифеста → предупреждение «нечего удалять»

---

## 2. Детекция хостов

### 2.1 Нормальные сценарии
- [ ] Оба хоста установлены → оба детектированы
- [ ] Только OpenCode → детектирован только он
- [ ] Только Claude Code → детектирован только он

### 2.2 Граничные случаи
- [ ] Ни один хост не установлен, `--hosts` не указан → ошибка «поддерживаемые AI-инструменты не обнаружены»
- [ ] Ни один хост не установлен, но указан `--hosts opencode` → попытка установки (хост может не существовать)
- [ ] Директория хоста существует, но пустая → детектируется как установленный
- [ ] Директория хоста существует, но нет прав на чтение → детектируется, но установка падает

---

## 3. Интерактивный TUI

### 3.1 Выбор хостов
- [ ] Multiselect с предвыбранными хостами → подтверждение выбора
- [ ] Снятие всех хостов → `required: true` не даёт продолжить
- [ ] Нажатие Ctrl+C → отмена, graceful exit

### 3.2 Выбор фич
- [ ] Multiselect с фичами → подтверждение
- [ ] Снятие всех фич → `required: false` позволяет продолжить
- [ ] Нажатие Ctrl+C → отмена

### 3.3 Uninstall confirmation
- [ ] `p.confirm` → подтверждение → удаление
- [ ] `p.confirm` → отказ → «Отменено»
- [ ] `p.confirm` → Ctrl+C → отмена

---

## 4. Установка файлов

### 4.1 Нормальные сценарии
- [ ] Копирование skills/ в целевую директорию хоста
- [ ] Manifest записывается в `~/.qarness/manifest.json`
- [ ] Manifest содержит правильные версии, даты, список файлов
- [ ] Повторная установка — обновление существующего манифеста (не затирает)

### 4.2 Граничные случаи — исходные файлы
- [ ] `skills/` не существует → пропуск без ошибки
- [ ] `skills/` пустая → нет файлов для копирования
- [ ] `agents/` не существует → пропуск
- [ ] Файл в skills/ с пробелами в имени → корректное копирование
- [ ] Файл в skills/ со спецсимволами → корректное копирование
- [ ] Символическая ссылка в skills/ → поведение (копируется или пропускается)

### 4.3 Граничные случаи — целевая директория
- [ ] Целевая директория не существует → создаётся
- [ ] Целевая директория уже содержит файлы с теми же именами → перезапись
- [ ] Нет прав на запись в целевую директорию → ошибка + rollback
- [ ] Диск заполнен → ошибка при копировании + rollback

### 4.4 Rollback
- [ ] Ошибка при копировании второго файла → удаление уже скопированных (reverse order)
- [ ] Ошибка в postInstall → rollback файлов + исключение
- [ ] Rollback при частично созданной директории → rm с `force: true`

---

## 5. Post-install модификация конфигов

### 5.1 OpenCode (opencode.json / opencode.jsonc)
- [ ] Конфиг существует, валидный JSON → добавляются paths и mcpServers
- [ ] Конфиг существует, невалидный JSON → пропуск (continue)
- [ ] Конфиг не существует → пропуск
- [ ] Конфиг уже содержит agents.paths → QArness-путь добавляется, не дублируется
- [ ] Конфиг уже содержит skills.paths → аналогично
- [ ] Конфиг уже содержит mcpServers.xmind → перезаписывается

### 5.2 Claude Code (settings.json)
- [ ] settings.json не существует → создаётся новый
- [ ] settings.json существует, валидный → добавляется mcpServers.xmind
- [ ] settings.json существует, невалидный → начинается с `{}`
- [ ] settings.json уже содержит mcpServers.xmind → перезаписывается

---

## 6. Фичи

### 6.1 xmind-mcp
- [ ] npm доступен → `npm install -g xmind-mcp-server` → успех
- [ ] npm не доступен → «skipped (npm not found)»
- [ ] npm install падает (сеть, права) → сообщение об ошибке
- [ ] Фича устанавливается только если есть успешно установленные хосты

### 6.2 Неизвестные фичи
- [ ] `--features nonexistent` → предупреждение, продолжение

---

## 7. Manifest

### 7.1 Чтение
- [ ] Манифест не существует → `readManifest()` возвращает `null`
- [ ] Манифест существует, валидный JSON → возвращается распарсенный объект
- [ ] Манифест существует, невалидный JSON → `null` (try/catch)
- [ ] Нет прав на чтение манифеста → ошибка (не обработано)

### 7.2 Запись
- [ ] `~/.qarness/` не существует → создаётся через `mkdir -p`
- [ ] Манифест содержит все поля: version, installedAt, hosts, features
- [ ] JSON pretty-printed (2 пробела) + trailing newline

### 7.3 Удаление
- [ ] `removeManifest()` → файл удаляется
- [ ] Файл уже не существует → пропуск (existsSync check)

---

## 8. Удаление (uninstall)

### 8.1 Нормальный сценарий
- [ ] Манифест существует → удаление файлов хостов + фич + манифеста
- [ ] Файлы из манифеста с trailing `/` → `rm -rf` (директория)
- [ ] Файлы из манифеста без `/` → `unlink` (файл)

### 8.2 Граничные случаи
- [ ] Манифест не существует → «Манифест не найден. Нечего удалять.»
- [ ] Файлы из манифеста уже удалены → `catch(() => {})` — тихий пропуск
- [ ] Файлы из манифеста заняты процессом → ошибка rm (не обработано)
- [ ] Unknown хост в манифесте → удаление по hostData без HostConfig (postUninstall не вызывается)

### 8.3 Post-uninstall
- [ ] OpenCode: QArness-пути удаляются из agents.paths, skills.paths, mcpServers.xmind
- [ ] Claude: mcpServers.xmind удаляется из settings.json
- [ ] Конфиг не существует → пропуск

---

## 9. CLI-команды (`src/cli/commands/`)

### 9.1 status
- [ ] Манифест существует → показывает версию, дату, хосты, фичи
- [ ] Манифест не существует → «QArness не установлен»

### 9.2 update
- [ ] Git доступен, репо — `git pull` → успех
- [ ] Git не доступен → ошибка
- [ ] Нет сети → git fetch падает
- [ ] Нет прав на `.git` → ошибка

### 9.3 integrate
- [ ] `qarness integrate opencode` → интеграция OpenCode
- [ ] `qarness integrate claude` → интеграция Claude Code
- [ ] `qarness integrate unknown` → показывает доступные AI-инструменты
- [ ] Указанный AI не детектирован → сообщение

### 9.4 config
- [ ] `config proxy set http://proxy:8080` → сохраняет прокси
- [ ] `config proxy clear` → удаляет прокси
- [ ] `config show` → показывает текущую конфигурацию
- [ ] `config proxy set` без URL → показывает usage
- [ ] Нет манифеста → создаётся новый

### 9.5 doctor
- [ ] Все проверки проходят → «Всё в порядке»
- [ ] Файлы отсутствуют → показывает проблему
- [ ] Зависимости не найдены → показывает рекомендации
- [ ] Манифест невалидный → показывает проблему

### 9.6 remove
- [ ] Манифест существует → удаление без подтверждения (отличие от uninstall.ts)
- [ ] Манифест не существует → «Не установлен»

---

## 10. Утилиты

### 10.1 commandExists
- [ ] Команда существует → `true`
- [ ] Команда не существует → `false`
- [ ] bash не доступен → `false` (try/catch)

### 10.2 copyDir / copyFile
- [ ] Исходная директория не существует → `mkdir` для dest, но `cp` упадёт
- [ ] Целевая директория уже существует → перезапись

### 10.3 listItems
- [ ] Директория существует с файлами → возвращает список
- [ ] Директория не существует → пустой массив
- [ ] С pattern → фильтрация по regex
- [ ] С exclude → исключение элементов
- [ ] Пустая директория → пустой массив

### 10.4 xdgConfig
- [ ] `XDG_CONFIG_HOME` задан → используется
- [ ] `XDG_CONFIG_HOME` не задан → `~/.config`

### 10.5 VERSION
- [ ] `.version` существует → читается и триммится
- [ ] `.version` не существует → `"unknown"`

---

## 11. Платформы

### 11.1 Linux/macOS
- [ ] `xdgConfig()` → `~/.config`
- [ ] Пути с прямыми слешами
- [ ] Права `chmod` при необходимости

### 11.2 Windows
- [ ] `homedir()` → `C:\Users\<user>`
- [ ] Пути с обратными слешами
- [ ] Named pipes для Docker в тестах
- [ ] `commandExists` через bash (Git Bash / WSL)

---

## 12. Параллельность и целостность

- [ ] Два экземпляра install.ts одновременно → конфликт записи манифеста
- [ ] Установка прервана (Ctrl+C) на середине → частичные файлы, манифест не записан
- [ ] Манифест записан, но postInstall упал → манифест не откатывается
- [ ] Удаление прервано → частично удалённые файлы, манифест уже удалён

---

## Как верифицировать

1. **Unit-тесты** (без Docker): `cli.ts` parseArgs, `utils.ts` listItems/commandExists, `manifest.ts` read/write
2. **Docker-тесты**: полный цикл install → verify → uninstall → verify clean
3. **Ручные сценарии**: TUI промпты, Ctrl+C, права доступа, заполненный диск
