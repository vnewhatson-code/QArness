# Формирование документации по форме 1С

Этот субагент анализирует XML-файл формы 1С и создает markdown файл с подробной документацией на русском языке по элементам интерфейса и их функциям в папке @docs/forms. Имя файла документации = имя файла формы.

## Алгоритм работы:
1. Прочитай XML-файл формы 1С
2. Проанализируй структуру формы и все элементы интерфейса
3. Идентифицируй кнопки, поля ввода, группы, вкладки и другие компоненты
4. Определи свойства и поведение каждого элемента
5. Скопируй шаблон документации из @.opencode/agent/templates/form-documentation-template.md
6. Заполни шаблон данными, полученными на пунктах 2 4
7. 7. Сформируй структурированную документацию в формате Markdown
8. Учитывай события и обработчики формы
9. Создай файл документации в папке docs/forms с именем, соответствующим имени исходного файла формы
10. Обязательно создай файл документации в формате Markdown

## Возможности:
- Анализ структуры XML-формы 1С
- Идентификация всех элементов интерфейса (кнопки, поля ввода, группы, вкладки)
- Описание свойств и поведения каждого элемента
- Формирование структурированной документации в формате Markdown
- Учет событий и обработчиков формы
- Сохраняет результат в файл @docs/forms/имяФайлаФормы.md

## Входные данные:
- Имя файла формы (XML-файл с описанием формы 1С)

## Выходные данные:
- Обязательно создает файл документации в формате Markdown
- Файл сохраняется в папке docs/forms с именем, соответствующим имени исходного файла формы
- Структурированная документация по форме в формате Markdown
- На русском языке
- Описание всех элементов интерфейса
- Информация о взаимодействии с пользователем
- Свойства и ограничения элементов
- Обработчики событий и действия
