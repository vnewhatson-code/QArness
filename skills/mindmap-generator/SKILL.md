# Mind Map Generator (simple mode)

## Назначение

Скилл строит майндкарту только из чеклиста:
- вход: `docs/QA/[feature_name]/CHECKLIST.md`
- выход: `docs/QA/[feature_name]/MINDMAP.md` и `docs/QA/[feature_name]/mindmap.xmind`

## Обязательные правила

1. Единственный источник данных: `CHECKLIST.md`.
2. Нельзя добавлять проверки, которых нет в чеклисте.
3. Весь текст результата должен быть на русском языке.
4. Если найдено несколько чеклистов и пользователь не указал папку, нужно остановиться и попросить выбрать папку.
5. `mindmap.xmind` создавать только после явного подтверждения пользователя.

## Запрещено

1. Не использовать `docs/QA/__workspace_inbox__/` как источник для генерации mind map.
2. Не использовать коммиты и исходный код как основной источник при наличии чеклиста.
3. Не продолжать работу при неоднозначном выборе чеклиста.
4. Не создавать `.xmind` без подтверждения пользователя.

## Алгоритм (строго по шагам)

### Шаг 1. Найти чеклисты

Искать файлы по маске: `docs/QA/*/CHECKLIST.md`.

### Шаг 2. Выбрать целевой чеклист

Использовать строго одну ветку:

- Если пользователь явно указал папку фичи и в ней есть `CHECKLIST.md` -> выбрать ее.
- Если найден ровно один `CHECKLIST.md` -> выбрать его.
- Если найдено несколько `CHECKLIST.md` и папка не указана -> остановиться с ошибкой E2.
- Если пользователь указал папку, но `CHECKLIST.md` там нет -> остановиться с ошибкой E3.
- Если не найден ни один `CHECKLIST.md` -> остановиться с ошибкой E1.

### Шаг 3. Прочитать и распарсить чеклист

Из выбранного `CHECKLIST.md` извлечь:
- название фичи;
- заголовки разделов;
- пункты проверок (включая `[ ] ...` и обычные буллеты);
- негативные и граничные сценарии (если есть в формулировках).

Удалить явные дублировки пунктов с одинаковым смыслом.

### Шаг 4. Создать `MINDMAP.md`

Создать/обновить файл:
`docs/QA/[feature_name]/MINDMAP.md`

Заполнять только на основе данных из чеклиста:
- разделы чеклиста -> категории майндкарты;
- пункты чеклиста -> ветки/листья;
- условия и ожидаемые результаты -> только если явно есть в тексте чеклиста.

### Шаг 5. Согласовать с пользователем

После генерации `MINDMAP.md` задать один вопрос:

`MINDMAP.md готов. Внести правки или создать mindmap.xmind?`

- Если пользователь просит правки -> внести правки в `MINDMAP.md` и снова задать тот же вопрос.
- Если пользователь подтверждает создание -> перейти к шагу 6.

### Шаг 6. Создать `mindmap.xmind` через MCP

Создать файл:
`docs/QA/[feature_name]/mindmap.xmind`

Источник: `MINDMAP.md` выбранной фичи.

## Фиксированные ошибки (использовать дословно)

- E1: `Не найден CHECKLIST.md. Сначала создайте чеклист в docs/QA/[feature_name]/CHECKLIST.md.`
- E2: `Найдено несколько CHECKLIST.md. Укажите папку фичи в docs/QA/.`
- E3: `В указанной папке нет CHECKLIST.md: docs/QA/[feature_name]/CHECKLIST.md.`
- E4: `Не удалось создать mindmap.xmind через MCP. Проверьте доступность MCP. MINDMAP.md сохранен.`

## Формат результата

После успешного завершения вернуть:
- путь к `MINDMAP.md`
- путь к `mindmap.xmind` (если создавался)

## Мини-проверка качества перед ответом

1. Использован только один выбранный `CHECKLIST.md`.
2. Все ветки майндкарты соответствуют пунктам чеклиста.
3. Нет явных дублей.
4. Текст на русском.
5. При нескольких чеклистах без выбора папки выдана ошибка E2.
