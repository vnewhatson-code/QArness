# Быстрый старт: checklist-generator

## Активация скилла

Скилл `checklist-generator` автоматически активируется, когда вы запрашиваете создание чеклиста для тестирования.

## Примеры команд для активации

### Вариант 1: С аналитикой и номером задачи
```
Создай чеклист для задачи DDOCS-1234
```

### Вариант 2: Только с аналитикой
```
Создай чеклист по файлу из inbox
```

### Вариант 3: Только по номеру задачи
```
Создай чеклист по задаче TASK-5678
```

## Подготовка данных

### Особенности (обязательно для учета на каждом запуске):
1. Используйте папку `docs/Speciality/`
2. В папке должен быть только один markdown-файл: `docs/Speciality/SPECIALITY.md`
3. Скилл всегда читает `SPECIALITY.md` перед генерацией чеклиста
4. Если в `docs/Speciality/` есть другие `.md` файлы, генерация останавливается с ошибкой

### Если используете файл аналитики:
1. Поместите файл с описанием фичи в `docs/QA/__workspace_inbox__/`
2. Файл может быть в формате .txt или .md
3. Включите описание функциональности, требования, сценарии

### Если используете номер задачи:
1. Убедитесь, что номер задачи присутствует в коммитах
2. Формат может быть: TASK-123, DDOCS-456, [PROJ-789], #123
3. Скилл найдет все коммиты с этим номером и проанализирует изменения

## Процесс работы

1. **Скилл определит режим работы** на основе доступных данных
2. **Проверит и прочитает** `docs/Speciality/SPECIALITY.md`
3. **Проанализирует** файл аналитики и/или git коммиты
4. **Запустит @explore** для анализа исходного кода
5. **Создаст** структурированный чеклист в `docs/QA/[название_фичи]/CHECKLIST.md`
6. **Предложит внести правки** если требуется

## Результат

Вы получите файл `CHECKLIST.md` со следующей структурой:
- Общая информация о задаче
- Функциональные проверки
- Проверки UI/UX
- Граничные условия
- Негативные сценарии
- Проверки на основе анализа кода (если доступен git)
- Регрессионные проверки
- Нефункциональные проверки

## Отличия от mindmap-generator

| Характеристика | checklist-generator | mindmap-generator |
|---|---|---|
| Выходной файл | CHECKLIST.md | MINDMAP.md + mindmap.xmind |
| Работа с git | ✅ Да | ❌ Нет |
| Режимы | 3 режима | 1 режим |
| Формат | Плоский чеклист | Иерархическая mind map |
